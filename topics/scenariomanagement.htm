Scenario Management

 

 

 

 

The "scenario Manager" interface can be accessed by clicking on "Scenario Manager" from Precept Main User Interface.

 

When working on Precept, there are three levels of hierarchy:

 1. Asset

 1. Project

 2. Scenario

 

A user can work on one or several Assets.

Users can create as many projects as needed under an Asset

There can be an unlimited number of scenarios under each Project.

 

 

The name of a scenario is the combination of the Asset Name + Project name  + scenario name

 

The screenshot below shows that for the Asset named 'Demo - Asset', under the project "Oct 23", 3 scenarios have been created:

 - Base

 - working forecast

 - Alternative scenario

 

 

The "Scenario Manager Interface allows users to:

 - create new Asset

 - create new Project

 - create new Scenario

 

 

Create new Asset

By default, when creating a new Asset, historical production allocation data have to be stored in Precept database. In case User wants to direcly connect to an existingproduction database, some configuration is needed. The connection string to the production allocation database and the required queries are stored in the table [Asset config] of Precept database. Please contact Cleverfield to ensure correct integration.

 

By default, when creating a new Asset, only "Super User" and the User who created the new Asset can access it. Additional users can be added using the "Security" interface (see Security / Access policy)

 

 

Create new Scenario

When creating a new scenario , the below interface will pop-up. User is then required to enter a name for the scenario, start and end date of the forecast.

 

 - duplicate scenarios

 - open the "Compare scenario" interface

 - delete scenario

 

 

When a scenario is created by a user, the user can decide to:

- share or not the scenario with other users

- set the status of the scenario to "working" or "locked"

 

 

Example

UserB can modify a scenario created by UserA if:

 - UserA has activated the "Share scenario" option

 - the status of the scenario is "working"

 

When the status of a scenario is locked, only the results of the scenario can be visualized. It is not possible to change the inputs or rerun a forecast.

 

 

Database Maintenance

Forecasting generates a lot of data and the database can grow in size quickly.

To keep good performance, it is important to keep a good maintenance of the database. "super-User" should click on the "Clean database" button once a month which will run an internal database cleaning process
