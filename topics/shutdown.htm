Shutdowns

 

 

 

 

Shutdown planning can be implemented by clicking on the 'Shutdown" icon on Precept main User Inferface.

 

Shutdown must be grouped by "Project". The project is used when calculating shortfalls. A project can also be added or not when generating forecast.

 

 

The table below shows a simple shutdown planning.

 

For example, there are 4 Shutdown projects in this dummy model:

 - Reservoir Management: used to shut wells for activities such as interference tests between wells

 - Well Hook-up: used to keep the wells shut-in few days after end of drilling to take into account well hook-up time

 - Maintenance

 - drilling interference: used to shut wells while drilling nearby wells

 

"Node SI" can be either a Well or a Node in the production Network.

 

Start and End Date of shutdowns can be defined using "hard coded" date or relative date. For modeling purposes and to be able to generate what-if scenario easily, relative should be used as much as possible. That what way, when a sequence of events (Drilling schedule, stimulation schedule, etc...) is shifted, there is no need to update the shutdown table.

Recurrent shutdowns can be implemented using the columns "Apply every x days" or "Apply every x months". The recurrent shutdowns can be stopped once date has passed "Apply recurrent shutdown until". If "Apply recurrent shutdown until" is left blank, the recurrent shutdowns will occur until end of forecast.
