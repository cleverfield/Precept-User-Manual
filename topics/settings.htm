Settings

 

 

 

 

Before running a forecast, Users may want to go to the Settings section to set up important parameters.

 

Objective Function:

Precept uses a linear solver to solve the network at every time step. This means some wells may be closed or choked-back to honor process constraints.

The wells that are closed or choked-back depend on the objectives function.

 

For example, on an oil field mostly gas constrained with the objectives to optimize oil production, wells with high GOR will be choked back when gas process constraints are hit.

 

Objective function is a not so easy function to set-up. This will normally be part of the initial modellling done by Cleverfield prior to install Precept. Should you wish to change the objective function, please contact Cleverfield.

 

 

 

Time step:

By default, Precept adds a time step whenever an event that will impact the forecast occurs. That means a time step will be added when:

- a new well come online,

- a well is converted from producers to injectors, or injectors to producers

- a shutdown starts /ends

- a new constraints is introduced in the forecast.

 

If none of the events above happen, a timestep is created every month as a minimum, unless the input "yearly time step after" is not blank. In that case, a timestep will be created every year as a minimum.

 

At every time step, potentials of wells are calculated from the DCA inputs (see Decline curves: Production and Injection potentials )

 

 

 

Future Losses

if that option is activated, Precept will not only calculate future production, but also future shortfalls. Shortfalls are grouped by Project.

 

 

 

Flush production

If that option is activated, flush/ transient production will be taken into account, meaning that after a shutdown, a producer will restart at a higher rate. If that option is not activated, the production before and after the shutdown will be the same. That option requires additional timesteps and therefore takes more tiem to run. That option should therefore be activated only for short-term forecast.

 

 

 

Voidage

if that option is activated, Precept will calculate production and injection per Patterns at reservoir conditions using the PVT data entered in the PVT section (PVT)

 

 

GHG emission:

if that option is activated, Precept will calculate GHG emission, provided that GHG inputs havebeen entered (GHG emission)

 

 

Economics:

if that option is activated, Precept will calculate Economics.

 

 

What-if scenario:

Users can generate what-if scenario by toggling on/off:

 - projects

 - surface constraints: unconstrained forecast can be generated if that option is off

 - uptime: forecast without uptime can be generated if that option is off.

 

 

Uptime

Uptime can be set at 3 levels.

 - Field level: from the settings input.

 - Manifold level: manifold are nodes directly downstream of a producer, or directly upstream of an injector. Uptime for manifold can be set from the "Constraints changes vs time" input interface. ( see Constraints changes vs time )

- Well level: Uptime for well can be set from the "Constraints changes vs time" input interface. ( see Constraints changes vs time )

 

Therefore:

At every time step, the total uptime for a well is:  Total Uptime for a well  = Field uptime x Manifold Uptime x Well Uptime

 

and:

At every time step: Actual Production = Choke opening x Potential from DCA curves,  with choke opening <= Total Uptime for the well

 

Once Precept solver has solved the network, "choke opening" can be less than "Total uptime" in case a well is process constraints.

Choke opening can be 0 if nodes located downstream are closed.
