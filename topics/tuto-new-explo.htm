Create new Exploration Asset and generate forecasts

 

 

 

 

1. Create a New Asset

From Precept Main User Interface, go to 'Scenario Manager", then click on "Create New Asset".

Give the name to the Asset (Brazil in our example) and click on "Create Asset".

 

 

 

 

 

2. Load the new Asset "Brazil" from Precept Main User Interface

 

 

 

At that point, only you and a "Super User" can access that Asset. Should you wish to allow more users, you need to add them via the "Security" user interface. (see Security / Access policy)

 

 

 

3. Create a new Project

The Asset does not have any project yet. A new project needs to be created. Go to 'Scenario Manager", then click on "Create New Project". We will call the new project "Evaluation"

Select the project "Evaluation".

The project "Evaluation" does not have any scenario yet.

 

 

4. Create a new scenario

Click on "Create new scenario". We will give it the name "Base". The option "share scenario with others users" is not activated.

 

 

5. Load the new scenario

The new scenario 'Brazil - Evaluation - Base" can now be loaded from Precept Main User Interface.

 

A new scenario created from scratch comes with a default "Network configuration" as shown below.

 

 

 

Default network configuration

 

 

6. Edit Network, Add constraints

For our Brazilian asset, we will keep that network configuration but network can be modified if needed (see Network modelling)

We will simply add a 120,000 bopd oil constrained to the node "Platform" to model a production plateau.

7. Define Wells potentials

Click on the "Decline curves" icon from Precept main User Interface to access Precept DCA module.

 

Oil producer OP1

Select the Asset "Brazil" , project "Evaluation" and Well "OP1".

Choose the type of decline ( see Decline curves: Production and Injection potentials)

 

Decline rate can be modified by drag-and_drop points on the chart or directly modify the value in the table.

Start point of declines can be modified by changing the values in the table.

 

In our case:

 Liquid at start: 25,000 stb/d

 Oil at start: 20,000 stb/d

 Gas at start: 5,000 stb/d

 Gas-lift: 0 mscf/d

 

Liquid nominal decline: 5%

Oil nominal decline: 10%

Gas nominal decline: 7%

 

b factor is 0 since exponential decline is selected.

 

Click "Save' to save the DCA for OP1

 

 

 

Water Injector WI1

in our example, we will use a constant WI potential of 5000 bwpd. Make sure the end date of the DCA (highlighted in red below) is enough to cover the full duration of the forecast.

Click "Save' to save the DCA for WI1

 

 

 

Gas Injector GI1

in our example, we will use a constant GI potential of 5000 mscf/d.

Click "Save' to save the DCA for GI1

8. Add a drilling sequence to drill the future wells

From Precept main User Interface, click on Sequence Add/ modify

We assume that a Rig called "Rig1" will start drilling 01/02/2024. Click save.

The column "Performed by" is not relevant for that tutorial and is only needed for the "autoschedule" option.

 

 

From Precept Main User Interface, click "Refresh", you can now see the Rig1 sequence, which is currently empty since it does not have any opportunities.

 

 

 

9. Create Opportunities to drill individual well

From Precept Main User Interface, click "Opportunities", then "New".

 

Opportunity to drill Oil producer OP1

 

The inputs below will create an Opportunity to drill the well OP1

Duration is set at 45 days.

The start date is set at "from sequence" meaning the opportunity start date is not "hard" but opportunity is going to be executed as part of a sequence.

Opportunity will be colored green in the drilling sequence.

 

Impact on forecast is set to "New profile. The new profile OP1 (which we defined in the DCA module) will come online once the opportunity OP1 is executed.

CAPEX is 30,000,000 USD

 

 

Duplicate this opportunity for all the other wells (OP2, OP3, GI1, WI1,..., GI1, etc..)

Click "Save" to create the 16 Opportunities.

 

 

After you click "Refresh" , you should see the 16 opportunities.

 

 

For visualization purposes, we can change the color (RGB column) for the opportunities to drill WI and GI wells.

 

Highlight the well "WI1" and click 'Edit". Change its color to blue and "Save"

Highlight the well "GI1" and click 'Edit". Change its color to Red and "Save"

 

Click "Refresh", you can now see that the column RGB for the WI1 has changed. Copy-paste (Ctr+C   Ctr +V) that value for other WI wells.

Click "Refresh", you can now see that the column RGB for the GI1 has changed. Copy-paste (Ctr+C   Ctr +V) that value for other GI wells.

The 16 Opportunities we created previously appears in the pool of Opportunity. Using mouse (right click and double-click), you can now move the Opportunities from the "pool" to the "Rig1" sequence. (see Build sequence of events)

 

It is proposed to drill the wells using the sequence below.

11. Define the objective function

The objective function is the value Precept solver tries to maximize at every timestep of the forecast.

 

From Precept Main User Interface click on the "Settings" icon.

The default Objective function is Maximize  "Oil_from_Platform_to_Oil_export" as shown below.

 

However, in our case, we also want to maximize water injection and gas injection. Therefore we need to modify that objective function

 

 

step 1: Select "Water Injection" at "WI" and click Add

The objective function has now been modified to include that Precept solver needs to maximize water injection at Node "WI". In "Precept language" it means maximize the value  "Water__Injection_from_Platform_to_WI"

 

 

step 2: Select "Gas Injection" at "GI" and click Add

The objective function has now been modified to include that Precept solver needs to maximize gas injection at Node "GI". In "Precept language" it means maximize the value  "Gas__Injection_from_Platform_to_GI"

 

 

 

 

11. Run the forecast

Set the forecast Start and End date and Click on "Run Forecast".

The timeline shows wells are put on production/injection once they are drilled.

Timeline indicates when the plateau is reached (row "Platform".

Timeline also highlights which wells are closed or choked-back when the 120,000 oil plateau is reached.

 

 

 

 

12. Visualize Results

From Precept main User Interface, click "Forecast charts" to visualize forecast at any nodes of the network
