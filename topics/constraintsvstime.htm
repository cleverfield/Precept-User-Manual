Constraints changes vs time

 

 

 

 

Initial process constraint can be set by opening the "Network" modeling (see section Shutdown)

 

Process constraints can be changed later in the forecast by two methods:

- executing an Opportunity ( see section Opportunities)

- "manually" change the value of a constraint at a specific date.

 

To manually change at a constraint a given time, Users need to click on the "Constraints vs Time" icon in Precept main interface.

 

 

In the above example, the following changes in constraints will be applied when generating forecast:

 

1. The max Water Injection rate of the well A1-I1 is set at 1000 bwpd between 28/10/2023 and 12/11/2023

 

2. The maximum Total Gas (Produced + Gas-lift) rate through "B_Compr" is set at 10000 mscf/d on 01/05/2023. That changes is applied every 12 months, meaning that Total Gas constraint will be set at 10,000 on 01/05/2024 , 01/05/2025, 01/05/2026, etc...

The maximum Total Gas rate through "B_Compr" is set at 12000 mscf/d on 01/11/2023. That changes is applied every 12 months, meaning that Total Gas constraint will be set at 12000 on 01/11/2024 , 01/11/2025, 01/11/2026, etc...

 

This method is used to introduce recurrent / seasonal capacity changes and will lead to the forecast shown below.

 

 

 

3. The well K-W3 uptime is set at 70%, from its first day of production, and for 7 seven days. Uptime to be back at 100% after.

 

 

 

Override solver

From this interface, Users can also use the option to "Override solver" for a well. This will force the well to not be shut-in or bean-back by Precept solver for process constraints.

This can be used to make sure a well is kept online even if Precept solver would like to shut-it to optimize hydrocarbons production.

 

 

 

Input can be QC using the timeline where any changes in contraints can be visualized.
